# daoSkilLs

## 目录用途

`daoSkilLs` 目录是 daoApps 生态系统中使用的技能的集中存储库。它提供了一种结构化的方式来组织、管理和维护各种技能，以增强 daoApps 平台的功能。

### 核心用途
- **技能管理**：将所有技能集中在一个位置，便于访问和维护
- **标准化**：确保所有技能具有一致的结构和文档
- **可重用性**：促进技能在 daoApps 平台不同部分的重用
- **可扩展性**：提供框架以支持平台增长时添加新技能

### 在项目中的角色
该目录作为 daoApps 的技能库，包含可调用以执行特定任务的专门功能。每个技能都设计用于处理特定的领域或用例，提供增强整体平台的专注能力。

## 主要功能

`daoSkilLs` 目录目前包含两个主要技能，每个技能都设计用于处理特定的用例：

### 1. alipay-payment-integration（支付宝支付集成）

**描述**：一个全面的技能，用于支付宝支付集成最佳实践，涵盖各种支付场景和集成方法。

**主要特性**：
- 支持多种支付场景：当面付、订单码支付、App支付、JSAPI支付、手机网站支付、电脑网站支付
- 为特殊场景提供指导，如预授权支付和商家扣款
- 包含安全指南和集成最佳实践
- 为不同支付产品提供模块化文档

**使用场景**：
- 将支付宝支付功能集成到应用程序中
- 为特定业务场景选择合适的支付方式
- 遵循支付集成的安全最佳实践

### 2. task-execution-summary（任务执行总结）

**描述**：一个用于生成任务执行过程结构化总结的技能，创建高质量的结构化经验报告。

**主要特性**：
- 使用10章标准模板自动生成全面的任务总结
- 支持不同详细程度：摘要版、标准版和详细版
- 基于任务分析提供智能建议
- 提供多种触发模式：自动、手动和基于命令

**使用场景**：
- Sprint回顾和项目评审
- 事故事后分析和故障排除文档
- 学习总结和知识管理
- 团队协作效果评估

## 文件结构

`daoSkilLs` 目录遵循结构化的组织方式，以确保一致性和易于维护。每个技能都有自己专用的目录，包含用于核心功能、文档和参考资料的标准化子目录。

### 目录树

```
daoSkilLs/
├── .git/                # Git 版本控制目录
├── .gitignore           # Git 忽略文件
├── LICENSE              # 许可证文件
├── README.md            # 本 README 文件
└── skills/              # 主要技能目录
    ├── alipay-payment-integration/  # 支付宝支付集成技能
    │   ├── SKILL.md     # 技能元数据和文档
    │   ├── modules/     # 模块化组件
    │   │   ├── core/    # 核心功能和指南
    │   │   ├── docs/    # 文档
    │   │   ├── products/ # 支付产品特定指南
    │   │   └── utils/   # 实用函数和工具
    │   └── references/  # 参考资料
    └── task-execution-summary/  # 任务执行总结技能
        ├── SKILL.md     # 技能元数据和文档
        ├── evals/       # 评估文件
        └── references/  # 参考资料和示例
```

### 目录说明

- **skills/**：包含所有技能的主目录。每个技能都有自己的子目录。
- **[skill-name]/SKILL.md**：每个技能都有一个 SKILL.md 文件，包含技能的元数据、描述和文档。
- **[skill-name]/modules/**：对于具有复杂功能的技能，模块提供了一种将代码和文档组织到逻辑组件中的方式。
- **[skill-name]/references/**：包含技能的参考资料、示例和附加文档。
- **[skill-name]/evals/**：包含用于测试和验证技能功能的评估文件。

### 关键文件

- **SKILL.md**：每个技能的主要文档文件，包含元数据、描述、使用说明和其他相关信息。
- **README.md**：本文件，提供 daoSkilLs 目录及其内容的概述。
- **LICENSE**：项目的许可证文件。

## 使用方法

### 通用使用模式

`daoSkilLs` 目录中的技能可以通过多种方式调用，具体取决于特定技能和使用上下文。一般来说，技能可以通过以下方式触发：

- **自然语言命令**
- **特定触发短语**
- **命令行调用**
- **程序化 API 调用**

### 使用 alipay-payment-integration 技能

**触发条件**：
- 当用户提到"接入支付宝"、"集成支付宝支付"、"对接支付"等
- 当用户讨论与支付相关的话题，如"支付宝收款"、"加个支付功能"、"支付宝下单"等
- 当用户提到特定支付方式，如"H5支付"、"小程序支付"、"预授权"等

**使用示例**：

```
# 示例 1：请求集成支付宝支付的指导
"我需要在我的应用中集成支付宝支付功能，应该如何操作？"

# 示例 2：询问特定支付方式
"如何实现支付宝的扫码支付功能？"

# 示例 3：寻求支付问题的帮助
"支付宝支付集成过程中遇到了签名验证失败的问题，如何解决？"
```

**输出**：
该技能将提供关于支付宝支付集成的全面指导，包括产品选择、集成步骤、安全指南和与用户特定需求相关的最佳实践。

### 使用 task-execution-summary 技能

**触发条件**：
- 当用户明确请求总结时，使用"总结"、"复盘"、"回顾"、"报告"等词
- 当用户完成任务并提到"完成了"、"结束了"、"搞定了"等
- 当用户问"做得怎么样"、"有什么收获"、"下次怎么做"等问题时

**使用示例**：

```
# 示例 1：完成任务后请求总结
"帮我总结一下这次bug修复的过程"

# 示例 2：请求 Sprint 回顾
"Sprint结束了，做个回顾吧"

# 示例 3：请求详细报告
"请生成详细版任务总结：网站性能优化"
```

**输出**：
该技能将生成一个包含多个部分的结构化总结报告，包括概述、目标、执行过程、关键决策、问题解决、资源使用、团队协作、分析、经验教训和改进措施。

**自定义选项**：
- **详细程度**：摘要版（2-3页）、标准版（8-15页）、详细版（20-30页）
- **模板类型**：标准版（通用）、学习版（用于教育目的）
- **语言风格**：专业版（正式）、随意版（非正式）

## 依赖项

### 通用依赖项

`daoSkilLs` 目录本身没有直接依赖项，但每个技能可能有自己的特定依赖项。以下是每个技能的依赖项：

### alipay-payment-integration 技能

**依赖项**：
- **支付宝开放平台 SDK**：用于与支付宝支付 API 集成
- **Node.js** (v12.0+)：如果使用 Node.js SDK 则需要
- **Java** (JDK 8+)：如果使用 Java SDK 则需要
- **PHP** (7.0+)：如果使用 PHP SDK 则需要
- **.NET** (4.5+)：如果使用 .NET SDK 则需要

**安装**：
```bash
# 对于 Node.js
npm install alipay-sdk

# 对于 Java（使用 Maven）
<dependency>
    <groupId>com.alipay.sdk</groupId>
    <artifactId>alipay-sdk-java</artifactId>
    <version>4.10.198.ALL</version>
</dependency>

# 对于 PHP
composer require alipaysdk/easysdk

# 对于 .NET
Install-Package Alipay.AopSdk.Core
```

**配置**：
- 支付宝商户账户
- 应用 ID 和私钥
- 支付宝公钥
- 支付回调的通知 URL

### task-execution-summary 技能

**依赖项**：
- **访问对话历史**：需要提取任务信息
- **文件系统访问**：需要保存生成的报告
- **Markdown 渲染能力**：需要生成格式化的报告

**安装**：
除了标准的 Trae IDE 环境外，不需要额外的依赖项。

**配置**：
- 不需要特定配置

## 开发指南

### 创建新技能

要在 `daoSkilLs` 目录中创建新技能，请按照以下步骤操作：

1. **创建新目录**在 `skills/` 下，使用描述性名称，采用短横线连接的小写格式（例如，`my-new-skill`）。

2. **创建 SKILL.md 文件**在技能目录中，使用以下结构：

```markdown
---
name: skill-name
id: skill-name
version: 1.0.0
description: >-
  技能的简要描述以及何时使用它。
dependencies:
  - dependency1
  - dependency2
---

# 技能名称

## 概述

[技能的详细描述]

## 使用方法

[如何使用该技能]

## 配置

[所需的任何配置]

## 示例

[使用示例]
```

3. **组织你的技能结构**：
   - 对于复杂的技能，创建子目录如 `modules/` 用于模块化组件
   - 使用 `references/` 存放参考资料和示例
   - 使用 `evals/` 存放评估文件

### 技能结构最佳实践

- **一致性**：遵循与现有技能相同的结构以保持一致性
- **模块化**：将复杂的技能分解为模块化组件
- **文档化**：在 SKILL.md 文件中记录技能的所有方面
- **示例**：提供清晰的使用示例
- **测试**：包含评估文件以测试技能的功能

### 命名约定

- **技能目录**：使用短横线连接的小写格式（例如，`alipay-payment-integration`）
- **文件名**：使用小写并用短横线分隔（例如，`skill-metadata.md`）
- **变量和函数**：使用驼峰命名法以保持一致性
- **文档**：使用清晰、描述性的标题和部分

### 测试和验证

要测试和验证你的技能：

1. **创建评估文件**在 `evals/` 目录中
2. **测试技能**使用各种输入以确保它正确响应
3. **验证输出**以确保它符合预期的格式和内容
4. **迭代和改进**基于测试结果

### 修改现有技能

修改现有技能时：

1. **阅读现有文档**以了解技能的工作原理
2. **进行增量更改**以避免破坏现有功能
3. **更新文档**以反映任何更改
4. **测试技能**以确保它仍然正常工作
5. **遵循现有的代码风格**和结构
